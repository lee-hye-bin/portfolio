/Front-End
  • + HTML

    HTML5 표준 문법과 웹 접근성을 준수하며 웹 페이지 개발에 임합니다. 시맨틱 태그를 활용해 사용자 친화적인 구조의 홈페이지를 설계하고 웹 페이지 각각의 섹션이 어떤 동작을 하는지 명확하게 전달합니다.

  • + CSS3

    CSS로 홈페이지의 디자인 요소를 조정할 수 있으며, CSS만을 사용하여 홈페이지 레이아웃을 구현할 수 있습니다. 또한 미디어쿼리를 통해 데스크탑뿐만 아니라 다양한 디바이스에 대응할 수 있는 안정적인 홈페이지를 제작함으로써 사용자에게 일관된 경험을 제공합니다.

  • + JavaScript

    기본기에 충실하기 위해 웹 페이지를 동적으로 제어할 수 있는 다양한 기능을 외부 플러그인 없이 구현할 수 있도록 학습했습니다. 외부 라이브러리를 사용할 때는 웹 페이지에 맞게 커스텀합니다. 이와 더불어 가독성이 좋고 유지보수하기 용이한 코드를 작성하기 위해 노력했습니다.

  • + SCSS

    CSS에서 확장된 언어인 SCSS를 활용해 코드의 유지보수를 쉽게 할 수 있습니다. 웹 페이지에서 반복적으로 사용될 변수를 미리 지정하고 작성한 스타일을 상속받아 코드 중복을 줄여 가독성을 높였습니다. 팀 프로젝트에서 사용한 언어인데, SCSS의 장점을 십분 활용해 팀 작업임에도 일관된 스타일을 유지할 수 있었습니다.

  • + React

    지금 공부 중인 라이브러리입니다. 따라서 능숙하게 사용할 수는 없지만, React를 사용해 동적인 웹 페이지를 구현할 수 있습니다. 다양한 곳에서 재사용되는 컴포넌트를 생명주기와 상태 메서드를 활용해 효과적으로 관리합니다.

학부생 때 백엔드 관련 언어와 프로그램을 배웠습니다. 졸업 이후 꾸준히 사용하지 않았기 때문에 능숙하지는 않지만, 백엔드에 대한 기본적인 개념과 원리를 알고 있습니다.

  • + PhotoShop

    간단한 이미지 편집과 그래픽 작업이 가능합니다. 이미지가 필요한 경우에는 포토샵을 이용해 빠르게 제작할 수 있습니다.

  • + Figma

    모든 프로젝트는 피그마에서 먼저 디자인을 진행한 후 개발했습니다. 피그마를 이용해 와이어 프레임을 만들고 스타일 가이드를 작성하며 웹 페이지의 프로토 타입을 효율적으로 제작했습니다. 또한, 팀 프로젝트에서 피그마를 사용해 공동으로 작업하며 팀원과 소통한 경험이 있습니다. 팀 작업을 할 때도 항상 스타일 가이드를 등록해 작업자가 달라져도 일관된 디자인을 유지할 수 있게 했습니다.

  • + Git

    프로젝트의 소스 코드와 버전 관리를 효과적으로 하기 위해 GitHub를 사용했습니다. GitHub를 통해 작업한 홈페이지를 호스팅하며 기본적인 Git의 기능을 익혔습니다.
